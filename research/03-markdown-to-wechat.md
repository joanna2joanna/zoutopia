# Markdown 写作 → 公众号发布：实操流程

> 2026-06-15 | 怎么在这里写，怎么发到公众号上

---

## 一、在 Claude Code 里写

### 写作环境
- 所有草稿放在 `drafts/` 目录下，`.md` 格式
- 使用 [template.md](../../drafts/template.md) 作为起点
- `published/` 存放已发布文章的归档（保留 Markdown 原文）

### Markdown 写作规范（跟公众号排版的对应关系）

| Markdown | 公众号呈现 |
|----------|-----------|
| `## 小标题` | 二级标题（16-18px，加粗） |
| `---` | 分隔线（视觉停顿） |
| `**加粗**` | 加粗强调 |
| `>` 引用 | 引导语/引言样式 |
| `![图片说明](path)` | 图片位置标记 |
| `<!-- 注释 -->` | 给自己看，公众号不显示 |

### 为什么 Markdown 足够

个人随笔不需要花哨排版。干净的 15px 正文 + 偶尔加粗 + 图片呼吸 = 最好的阅读体验。Markdown 的克制和公众号随笔的克制是同一件事。

## 二、排版转换（选一个）

### 方案 A：墨滴（推荐，最简单）

1. 打开 [mdnice.com](https://mdnice.com)
2. 把草稿全选复制，粘贴到左边编辑栏
3. 在主题列表里选一个**最干净的**（推荐"全栈蓝"或"极简"）
4. 微调——墨滴支持在主题基础上改 CSS，对应我们之前定的参数：

```css
/* 粘贴到墨滴的自定义CSS区域 */
p {
  font-size: 15px;
  color: #333333;
  line-height: 1.75;
  letter-spacing: 1px;
  margin-bottom: 0.5em;
}
```

5. 点右上角「复制到公众号」→ 打开公众号后台 → Ctrl+V 粘贴
6. 在公众号编辑器里预览手机效果，调整图片位置
7. 发布

### 方案 B：doocs/md

1. 打开 [doocs.github.io/md](https://doocs.github.io/md)
2. 同样的流程：粘贴 → 调主题 → 复制 → 到后台粘贴

### 方案 C：壹伴助手（如果主要用 Word/飞书）

1. 装 Chrome 插件「壹伴」
2. 公众号后台会出现「导入 Word」按钮
3. 支持 .docx 文件拖拽上传，保留排版

## 三、图片处理

### 对于走托邦，图片来源可能是：
- 自己拍的照片
- 喜欢的插画/截图（注意版权，个人随笔用截图引用一般可行）
- 纯色背景 + 一句金句（最简单的封面图做法）

### 操作步骤

1. **正文插图**在墨滴/doocs 里用 Markdown 的 `![alt](path)` 标记位置
2. 本地图片需要先上传——墨滴有"图片上传"功能，会自动替换为图床 URL
3. 或者直接在公众号编辑器里，在对应位置手动插入图片
4. **封面图**：在公众号后台单独上传，尺寸 900×383px（2.35:1）
   - 最简单的做法：Canva 选公众号封面模板 → 写标题 → 导出

## 四、完整发布检查清单

```
写作阶段
☐ 字数 800-1200
☐ 每段 ≤ 4 行
☐ 开头 3 句内进入场景
☐ 收尾句留了缝隙

排版阶段
☐ 15px / #333 / 1.75倍行距
☐ 段落之间空一行
☐ 图片前后各空一行
☐ 整篇颜色 ≤ 2 种（正文灰 + 1 个强调色）

发布前
☐ 手机预览一遍（在公众号后台点"手机预览"）
☐ 检查标题（有没有具体画面感？）
☐ 检查封面图（900×383，文字不被裁剪）
☐ 选"发布"还是"群发"？
    - 群发 = 推送通知给所有订阅者，每天限 1 次
    - 发布 = 不推送，但出现在主页，不限次数
```

## 五、走托邦发布节奏建议

起步阶段不用想太多：

- **第一周**：写 2-3 篇短随笔，用「发布」（不群发），先让主页有内容
- **第二周起**：挑一篇自己最满意的，群发给第一批好友
- **稳定频率**：先不设目标频率。写出来一篇就发一篇。积累 10 篇之后再来复盘节奏。

> 关键不是"多久发一次"，是"一直在发"。不等那个完美的第一篇。
